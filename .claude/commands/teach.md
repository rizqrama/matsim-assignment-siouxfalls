# /teach — Explain Before Building

Before proceeding with implementation, explain the following
to the user in beginner-friendly terms:

1. **What MATSim component is being used**
    - Name it and give a plain-language definition
    - Use an analogy if the concept is abstract

2. **Why this approach is chosen**
    - What alternatives exist
    - Why this one is preferred for this situation

3. **How it connects to real-world transport**
    - What transport planning concept does this represent
    - What would this look like in a real city or system

4. **What best practice this follows**
    - Why this pattern matters for larger or more complex projects
    - What problems it prevents down the line

5. **How this fits into the overall project structure**
    - Where does this piece sit in the full MATSim pipeline
    - What comes before it and what depends on it

Use simple language. Define any technical terms used.
Use analogies where helpful.
Only proceed to implementation after this explanation is complete.
